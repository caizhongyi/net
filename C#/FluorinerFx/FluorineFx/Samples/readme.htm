FluorineFx Samples Readme

The FluorineFx samples are simple applications that demonstrate the features of FluorineFx. Each sample typically consists of an ASP.NET (.NET2) Web site project, a service library project and the Flash/Flex client application.

To run these samples you need a copy of Visual Studio 2005(or Visual Web Developer 2005 Express Edition) and Adobe Flex Builder 2. Command line build files are not included with these samples.

The layout of the samples is:
For Flash clients - FluorineFx Installation directory\Samples\Flash\SampleFolder
For Flex clients - FluorineFx Installation directory\Samples\Flex\SampleFolder

Samples List

This is the list of the FluorineFx samples:

Flash - These samples demonstrate how Flash and Flash Remoting can be used with FluorineFx.

  • Remoting\Activation - Demonstrates how to configure object activation (scope). FluorineFx provides an extensible factory mechanism that lets you control your component creation and maintenance.
  • Remoting\Authentication - Demonstrates how to configure and enforce authentication and authorization for Flash Remoting.
  • Remoting\LegacySystem - Demonstrates how to use Flash Remoting without flex configuration files.
  • Remoting\Recordset - Demonstrates how to use Flash Remoting and Recordsets.
  • Remoting\WebService - Demonstrates how to call .NET web services from Flash using Flash Remoting.
  • RemotingAS3\RecordsetAS3LRF - Demonstrates how to use Flash Remoting from Flash CS3/AS3 using the AS3 Lightweight Remoting Framework.
  • RemotingAS3\RecordsetBA - Demonstrates how to use Flash Remoting from Flash CS3/AS3 using the ByteArray.org NetConnection wrapper.
  • Rtm - Real time messaging applications: remote shared object, publishing and recording stream, bandwidth detection samples
    Rtm\BWCheck
    Rtm\SharedBall This tutorial demonstrates how users can share a graphic object. You can move a ball around the screen and to watch while other participants move the same ball around. To view the application open two instances in a Web browser.
    Rtm\VideoChat
    Rtm\VideoRecording

Flex - These samples demonstrate how Flex can be used with FluorineFx.

  • Remoting\Authentication - Demonstrates how to configure and enforce authentication and authorization.
  • Remoting\MultipleClientAuthentication - Demonstrates how to configure and enforce authentication and authorization from a WebForm, Flash client and Flex client.
  • Remoting\ByteArray - Demonstrates how to work with ByteArrays.
  • Remoting\Registration - Captcha sample using ByteArray.
  • Remoting\DataAccess - Demonstrates how to access DataTables using Remoting.
  • Remoting\Caching - Demonstrates how to use service result caching.
  • Remoting\Externalizable - Demonstrates how to use IExternalizable.
  • Remoting\HttpCompression - Demonstrates how to configure http compression for remoting calls (amf channel only).
  • Remoting\WebService - Demonstrates how to call .NET web services from Flex.
  • Remoting\RemoteObjectAMF0 - Demonstrates how to use Renaun Erickson's RemoteObjectAMF0 component.
  • SharedObject\SharedBall - This tutorial demonstrates how users can share a graphic object. You can move a ball around the screen and to watch while other participants move the same ball around. To view the application open two instances in a Web browser.
  • SharedObject\BouncingBalls - Another tutorial that demonstrates how users can share a graphic object. To view the application open two instances in a Web browser.
  • SharedObject\Chat - This tutorial demonstrates how to use shared objects to create a simple text chat application. To view the application open two instances in a Web browser.
  • SharedObject\StressTest - Simple stress test (based on Red5 SOStress by Nathan Smith).
  • Rtm\VideoChat - Flex VideoChat sample.
  • Rtm\WebRadio - Flex and Flash server streaming sample.
  • Messaging\Chat - This tutorial demonstrates how to use the publish-subscribe message flow (Producer/Consumer) to create a simple text chat application. To view the application open two instances in a Web browser.
  • Messaging\DateFeed - This tutorial demonstrates how to push messages from the server to the client.
  • Messaging\WebMessenger - This tutorial demonstrates a simple web based messenger, based on the sample by Christophe Coenraets.
  • Messaging\MarketFeed - This tutorial demonstrates how to use subtopics and selector expressions in messaging applications.
  • MSMQ\OrderProcessing - This tutorial demonstrates how to integrate MSMQ, FluorineFx with Flex messaging.
  • DataServices\CRM - This tutorial demonstrates using the Flex Data Services 2 CRM sample application with FluorineFx.
  • Spring - This tutorial demonstrates how to use Spring.NET with FluorineFx.
  • MS Enterprise Library - This tutorial demonstrates how to use MS Enterprise Library/Policy Injection with FluorineFx.
  • Optimizer - This tutorial demonstrates how to configure reflection optimizer for AMF deserialization.

Air - These samples demonstrate how Adobe Integrated Runtime (Adobe AIR™) can be used with the FluorineFx Windows Service.

  • Northwind - Demonstrates how to build a simple application and connect to the Northwind database. The sample application uses the FluorineFx Windows Service.
  • Stats - Demonstrates how to build a simple application and query computer information. The sample application uses the FluorineFx Windows Service.

Ajax - These samples demonstrate how JavaScript clients in web pages can be used with the FluorineFx.

  • DataAccess - Demonstrates Ajax integration.

Steps for Working with the Samples

  1. Open the Visual Studio 2005 solution file located in the sample folder.
  2. Read the readme.htm file associated with the sample for additional information (if available).
  3. Start the application with the "Start without Debugging" command (Ctrl+F5)

Configuring the Client Applications

  1. Flex projects using LiveCycle Data Services features (fds.swc) are configured to use an Eclipse Linked Resource pointing to "[LiveCycle Data Services Install Dir]\jrun4\servers\default\flex\WEB-INF\flex" (lcds_webapp_root/WEB-INF/flex). The path variable name is "LCDS".
