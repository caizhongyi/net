Reflection optimizer

Demonstrates how to configure reflection optimizer for AMF deserialization.

Reflection optimizer is turned on by setting the optimizer tag in the web.config file.

<!-- Reflection optimizer provider="codedom|il" debug="true|false" -->
<optimizer provider="codedom" debug="true"/>

Supported options are the CodeDOM-based optimizations (this is the only option for .NET 1.1) and Lightweight Code Generation (LCG) based optimizations (available only for .NET 2.0).
Debugging is available only for CodeDOM-based optimizations.

Note1: Reflection optimizer cannot be used with nullable types (and null/undefined values sent for .NET primitive types are not supported).

Note2: Special considerations are required when working with AMF0.

Note3: This is an advanced setting, if the application behaves incorrectly turn off the reflection optimizer.
